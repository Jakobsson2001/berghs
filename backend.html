What is a Backend?

Think of a website like a restaurant. The frontend is what customers see ‚Äî the menu, the dining room, the plates. The backend is the kitchen ‚Äî where orders are processed, ingredients are stored, and food is prepared. Customers don't see the kitchen, but it's essential for the restaurant to function.

In web development, the backend handles:

  • Storing data ‚Äî User accounts, posts, products, anything that needs to be saved
  • Processing requests ‚Äî When a user submits a form or clicks a button, the backend decides what to do
  • Security ‚Äî Protecting sensitive information and validating user actions
  • Business logic ‚Äî Calculations, validations, and rules that make your app work

Backend Installation Guide

Before you can start building backends, you need to set up your development environment. This guide covers installing Python (for Flask/FastAPI) and setting up databases.

Installing Python (Recommended for Beginners)

Python is an excellent choice for backend development. It comes pre-installed on macOS, but you may need to install or update it on Windows.

üì± macOS (MacBook)

  1. Python 3 is usually pre-installed. Check by opening Terminal and running: 
    python3 --version
  2. If Python is not installed or you want the latest version:
    • Visit python.org/downloads
    • Download the latest Python 3.x version for macOS
    • Run the installer and follow the wizard
    • Make sure to check "Add Python to PATH" if the option appears
  3. Verify installation: 
    python3 --version
pip3 --version

ü™ü Windows

  1. Visit python.org/downloads
  2. Download the latest Python 3.x version for Windows
  3. Run the downloaded .exe installer
  4. Important: Check the box "Add Python to PATH" at the bottom of the installer
  5. Click "Install Now" and wait for installation to complete
  6. Open Command Prompt or PowerShell
  7. Verify the installation: 
    python --version
pip --version
    You should see version numbers for both commands.

Setting Up Your First Backend Project

Once you have Python installed, you can create your first backend project. We'll use Python with Flask, which is simple and beginner-friendly.

Python Flask Backend

Step 1: Create a Project Folder

# Create a new folder
mkdir my-backend
cd my-backend

Step 2: Create a Virtual Environment (Recommended)

Virtual environments keep your project dependencies isolated:

# macOS/Linux
python3 -m venv venv
source venv/bin/activate

# Windows
python -m venv venv
venv\Scripts\activate

Step 3: Install Flask

pip install flask

Step 4: Create Your First API

Create a file called app.py:

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/api/hello', methods=['GET'])
def hello():
    return jsonify({'message': 'Hello from backend!'})

if __name__ == '__main__':
    app.run(debug=True, port=5000)

Step 5: Run Your Server

python app.py

üí° Your API will be available at http://localhost:5000/api/hello

Setting Up Databases

For your backend project, you'll need a database to store data. Here are two great options for learning:

MongoDB (Recommended)

MongoDB is a flexible NoSQL database that's great for learning and real projects. It stores data as documents (like JSON), which is intuitive for beginners.

Option 1: MongoDB Atlas (Cloud - Free Tier)

  1. Visit mongodb.com/cloud/atlas
  2. Sign up for a free account
  3. Create a free cluster (M0 - Free tier)
  4. Get your connection string from the "Connect" section
  5. Install the Python MongoDB driver: pip install pymongo

Option 2: Local MongoDB Installation

You can also install MongoDB locally on your computer:

  • macOS: brew install mongodb-community
  • Windows: Download from mongodb.com/try/download/community

Python Example with MongoDB

from pymongo import MongoClient

# Connect to MongoDB
client = MongoClient('mongodb://localhost:27017/')
db = client['my_database']
collection = db['users']

# Insert data
collection.insert_one({'name': 'Alice', 'age': 30})

# Find data
users = collection.find({'name': 'Alice'})

SQLite (For Simple Projects)

SQLite is a file-based database ‚Äî no server setup needed! Perfect for learning and small projects. It comes built-in with Python.

Python (with SQLite)

# SQLite comes built-in with Python
import sqlite3

conn = sqlite3.connect('database.db')
cursor = conn.cursor()

# Create a table
cursor.execute('''CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT)''')

# Insert data
cursor.execute("INSERT INTO users (name) VALUES (?)", ('Alice',))

# Query data
cursor.execute("SELECT * FROM users")
users = cursor.fetchall()

conn.commit()
conn.close()

Choosing a Database

A database is like a digital filing cabinet where you store all your app's data. When you create a user account, post a comment, or save a product, that information goes into a database.

üìê Database Design in Real Projects:

In professional projects, teams typically create a UML (Unified Modeling Language) diagram or an Entity Relationship Diagram (ERD) to document the database structure before writing any code. This helps everyone agree on the data model, relationships between tables, and ensures the database design is well thought out. For this class, you can start simple, but it's good to know this is standard practice in real development.

Simple Python Backend Example

Here's a basic Flask server that returns JSON data:

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/api/users', methods=['GET'])
def get_users():
    users = [
        {'id': 1, 'name': 'Alice'},
        {'id': 2, 'name': 'Bob'}
    ]
    return jsonify(users)

if __name__ == '__main__':
    app.run(debug=True)

üí° This creates an API endpoint at http://localhost:5000/api/users that your frontend can call to get user data.

Connecting Frontend to Backend

There are several ways to connect your frontend (React, HTML, etc.) to your backend. The method you choose depends on your security needs and project complexity.

üèóÔ∏è Architecture Note: Monolith vs Microservices

In professional development, teams often debate between monolith (everything in one codebase) and microservices (separate services for different features). For this class, you can keep everything in the same project ‚Äî your frontend, backend API routes, and database connections can all live together. This is simpler to learn and manage, and perfectly fine for most projects. You can always split things later if needed!

Method 1: REST API (Most Common)

REST API is the standard way to connect frontend and backend. Your frontend makes HTTP requests (GET, POST, PUT, DELETE) to your backend, which responds with data (usually JSON).

How It Works

  1. Frontend sends a request: fetch('https://api.example.com/users')
  2. Backend processes the request and queries the database
  3. Backend sends back JSON data: { users: [...] }
  4. Frontend receives the data and updates the UI

Example: Fetching Data in React

// In your React component
function UserList() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    // Fetch data from your backend API
    fetch('http://localhost:5000/api/users')
      .then(response => response.json())
      .then(data => setUsers(data));
  }, []);

  return (
    <div>
      {users.map(user => (
        <p key={user.id}>{user.name}</p>
      ))}
    </div>
  );
}

Method 3: GraphQL (Advanced)

GraphQL is an alternative to REST that lets you request exactly the data you need. It's more flexible but also more complex. For this class, REST APIs are recommended.

Security Considerations (Optional)

When connecting frontend to backend, security is important ‚Äî especially if you're handling user data, payments, or sensitive information.

üîí Important Security Basics:

  • Never expose API keys in frontend code ‚Äî Use environment variables or backend-only storage
  • Validate data on the backend ‚Äî Don't trust data from the frontend
  • Use HTTPS ‚Äî Encrypt data in transit
  • Implement authentication ‚Äî Verify users are who they say they are
  • Use CORS properly ‚Äî Control which websites can access your API

‚úÖ For This Class:

For learning and simple projects, you can start with basic API calls. Security becomes more important as your projects grow. Always validate data on the backend and never trust input from the frontend.

Recommended Learning Path

Here's a suggested path for learning backend development:

Step 1: Start Simple

Build a Python backend with Flask and connect to SQLite:

  • Create a Flask server (see installation guide above)
  • Set up SQLite database (comes with Python)
  • Create API routes that read/write data
  • Test your API with a tool like Postman or curl

Step 2: Connect to MongoDB

Upgrade to MongoDB for more realistic projects:

  • Set up MongoDB Atlas (free cloud tier) or install locally
  • Connect your Flask backend to MongoDB using pymongo
  • Create REST API endpoints that interact with MongoDB
  • Connect your React frontend to this Python backend

Step 3: Understand Security

As your projects grow, learn about:

  • Authentication (JWT tokens, sessions)
  • Input validation and sanitization
  • Environment variables for secrets
  • CORS and API security
