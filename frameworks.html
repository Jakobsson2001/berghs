From Basics to Frameworks

Once you've learned the fundamentals of HTML, CSS, and JavaScript, you might notice that writing everything from scratch becomes repetitive. Each new project needs navigation bars, layouts, buttons, and similar patterns. Developers quickly realized they were "reinventing the wheel" ‚Äî building the same components again and again.

Why Use Frameworks?

Frameworks like React, Tailwind CSS, and Material UI (MUI) solve this problem by providing reusable building blocks and consistent patterns. Instead of focusing on boilerplate code, you can focus on logic, interactivity, and design. These tools help you write clean, maintainable code faster ‚Äî without starting over every time.

Installation Guide

Before you can start using React and other frameworks, you need to set up your development environment. This guide will walk you through installing the necessary tools on both macOS (MacBook) and Windows.

Step 1: Install Node.js and npm

npm (Node Package Manager) comes bundled with Node.js. It's the default package manager for JavaScript and allows you to install libraries and tools for your projects. npm is sufficient for this class ‚Äî you don't need to install any additional package managers.

üì± macOS (MacBook)

  1. Visit nodejs.org
  2. Download the LTS (Long Term Support) version for macOS
  3. Open the downloaded .pkg file and follow the installation wizard
  4. Open Terminal (found in Applications ‚Üí Utilities)
  5. Verify the installation by running: 
    node --version
npm --version
    You should see version numbers for both commands.

ü™ü Windows

  1. Visit nodejs.org
  2. Download the LTS (Long Term Support) version for Windows
  3. Run the downloaded .msi installer file
  4. Follow the installation wizard (accept the default settings)
  5. Open Command Prompt or PowerShell (search for "cmd" or "powershell" in the Start menu)
  6. Verify the installation by running: 
    node --version
npm --version
    You should see version numbers for both commands.

üì¶ About Yarn:

Yarn is optional for this class. While Yarn is faster and more secure, npm is sufficient and recommended for learning React. If you encounter permission issues on Mac and need to install Yarn, you can use sudo npm install -g yarn (see the troubleshooting section below for more details).

Step 2: Create Your First React Project

Once you have Node.js and npm installed, you can create a new React project. React provides an official tool called Create React App that sets up everything you need. We recommend following the official Create React App documentation.

Using npm (recommended for this class):

npx create-react-app my-first-app
cd my-first-app
npm start

üí° Replace my-first-app with your desired project name. The npm start command will open your React app in the browser at http://localhost:3000.

üí° Note: npx comes with npm and doesn't require global installation, so you don't need sudo for this command.

Understanding package.json

When you create a React project, a file called package.json is automatically created in your project folder. This file is the heart of your project ‚Äî it contains important information about your project and its dependencies.

What's in package.json?

Here's what a typical package.json looks like:

{
  "name": "my-first-app",
  "version": "0.1.0",
  "private": true,
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0"
  },
  "scripts": {
    "start": "react-scripts start",
    "build": "react-scripts build",
    "test": "react-scripts test"
  }
}
  • name: Your project's name
  • version: The current version of your project
  • dependencies: Lists all the libraries your project needs (like React, React DOM, etc.)
  • scripts: Commands you can run with npm run (e.g., npm start runs the "start" script)

Installing Additional Packages

When you install a new package (like Tailwind CSS or MUI), it gets added to your package.json:

npm install tailwindcss

üí° After installation, you'll see the new package listed in the dependencies section of your package.json. This file ensures that anyone who works on your project can install the exact same packages by running npm install.

‚úÖ You're all set! Once you've completed these steps, you have everything you need to start building React applications. The next section will explain how React works under the hood.

Why React Instead of Plain HTML/CSS?

You might wonder: "If HTML and CSS work fine, why learn React?" Here are the real problems React solves that become painful with plain HTML/CSS/vanilla JavaScript:

1. Managing Dynamic Content

With plain HTML, updating content means manually finding and changing DOM elements. React automatically updates the UI when your data changes.

Plain HTML/JS (Complex)
// Update a counter manually
const countEl = document.getElementById('count');
const button = document.getElementById('increment');
let count = 0;

button.addEventListener('click', () => {
  count++;
  countEl.textContent = count; // Manual update
  // What if count appears in 5 places?
  // You must update each one manually!
});
React (Simple)
// React automatically updates UI
function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}
// UI updates automatically everywhere!

2. Reusing Components

Need the same button or card in multiple places? With HTML, you copy-paste code. With React, you create a component once and reuse it everywhere. 

// React: Create once, use everywhere
function ProductCard({ name, price }) {
  return (
    <div className="card">
      <h3>{name}</h3>
      <p>${price}</p>
    </div>
  );
}

// Use it multiple times with different data
<ProductCard name="Laptop" price="999" />
<ProductCard name="Mouse" price="29" />
<ProductCard name="Keyboard" price="79" />

// With HTML, you'd write the same structure 3 times!

3. Keeping UI in Sync with Data

When data changes (like new messages, updated user info, or form submissions), React automatically re-renders only what changed. With plain JavaScript, you must manually track and update every affected element.

4. Building Complex Interactive UIs

Features like forms with validation, search filters, shopping carts, or real-time chat become exponentially more complex with vanilla JavaScript. React's component model and state management make these manageable.

üí° When to use what: Use plain HTML/CSS for simple static websites. Use React when you need interactivity, dynamic content, reusable components, or complex user interfaces. Most modern web applications benefit from React.

Understanding Package Managers

Before diving into React, it's important to understand package managers. A package manager is a tool that helps you install, update, and manage software libraries (called "packages" or "dependencies") that your project needs to run.

What is a Package Manager?

Think of a package manager like an app store for code. Instead of manually downloading and copying files, package managers automatically:

  • Download packages from online repositories (like npm registry)
  • Install dependencies ‚Äî packages your project needs to function
  • Manage versions ‚Äî ensure compatibility between different packages
  • Resolve conflicts ‚Äî handle when packages need different versions of the same dependency

Common Package Managers

Different package managers serve different purposes:

  • npm (Node Package Manager) ‚Äî Comes bundled with Node.js, used for JavaScript/Node.js projects. This is what we'll use for React.
  • yarn ‚Äî An alternative to npm, also for JavaScript projects. Faster and more secure, but not required for this class.
  • brew (Homebrew) ‚Äî A package manager for macOS/Linux that installs system-level software (like Node.js itself). Not for project dependencies.
  • apt/yum ‚Äî System package managers for Linux distributions (Ubuntu, Fedora, etc.)

üì¶ For This Class:

npm is sufficient and recommended. It comes with Node.js, so once you install Node.js, you have npm ready to use. You don't need to install yarn or any other package manager for React development in this course.

npm vs yarn: What's the Difference?

Both npm and yarn are JavaScript package managers, but they have some differences:

npm

  • Comes bundled with Node.js
  • Widely used and well-documented
  • Sufficient for most projects
  • Standard in the industry
  • Good for learning (simpler ecosystem)

yarn

  • Faster installation (parallel downloads)
  • More secure (checksums for packages)
  • Lighter footprint (better caching)
  • Requires separate installation
  • Optional for this class

For this class: Use npm. It's simpler, comes with Node.js, and is perfectly capable for learning React. Yarn is great for production teams where speed and security are critical, but it's not necessary for educational purposes.

Troubleshooting: Permission Issues on Mac

If you encounter permission errors when installing packages globally on macOS (like EACCES errors), you may need to use sudo. This is common on Mac systems where npm's global directory requires administrator privileges.

If you need to use sudo: 

# For installing yarn globally (if you choose to use it):
sudo npm install -g yarn

# For other global packages:
sudo npm install -g [package-name]

Alternative solutions (if you want to avoid sudo):

  • Use npx (recommended): You don't need to install packages globally. npx create-react-app works without global installation, so no sudo needed.
  • Fix npm permissions: Change npm's default directory to avoid permission issues: 
    mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
# Add this to your ~/.zshrc or ~/.bash_profile:
# export PATH=~/.npm-global/bin:$PATH
  • Use a Node version manager: Tools like nvm (Node Version Manager) install Node.js in your home directory, avoiding permission issues entirely.

‚ÑπÔ∏è About sudo:

Using sudo with npm is sometimes required on Mac systems. While it works, be aware that: (1) it gives npm root privileges (security consideration), and (2) it can sometimes cause permission conflicts later. For this class, using npx avoids the need for sudo entirely. If you do use sudo and encounter issues later, you can fix permissions using the methods above.

How React Works: The Foundation

React is a JavaScript library that builds upon the web standards you've already learned. At its core, React uses JSX (JavaScript XML), a syntax extension that looks like HTML but is actually JavaScript. When you write React components, you're describing the structure and behavior of your UI in a declarative way.

React's Relationship with HTML

React components compile down to standard HTML elements through a process called rendering. When React renders a component, it creates a Virtual DOM (a JavaScript representation of the HTML structure), then efficiently updates the actual browser DOM (the HTML you see in the browser) only where changes occur.

Here's how it works: You write JSX that looks like HTML, React processes it, and the browser receives standard HTML elements. Your knowledge of HTML semantics, accessibility attributes, and element structure directly applies in React. 

// React Component (JSX)
function Button() {
  return (
    <button className="btn-primary" onClick={handleClick}>
      Click me
    </button>
  );
}

// Renders to standard HTML:
// <button class="btn-primary">Click me</button>

React's Relationship with CSS and Tailwind

React components can use CSS in multiple ways, and this is where Tailwind CSS becomes powerful:

  • Standard CSS: You can import CSS files or use inline styles just like in traditional HTML. React doesn't change how CSS works.
  • Tailwind CSS: Tailwind's utility classes work seamlessly with React. You apply Tailwind classes directly to JSX elements using the className attribute (note: it's className in React, not class, because class is a reserved word in JavaScript).
  • CSS-in-JS: React also supports writing styles as JavaScript objects, but Tailwind's utility-first approach is often preferred for its simplicity and performance.
// React component with Tailwind CSS
function Card() {
  return (
    <div className="bg-white rounded-lg shadow-md p-6">
      <h2 className="text-2xl font-bold text-gray-900">
        Title
      </h2>
      <p className="text-gray-600 mt-2">Content</p>
    </div>
  );
}

// Tailwind classes compile to standard CSS
// The browser receives: <div class="bg-white rounded-lg...">
// Which applies: background-color: white; border-radius: 0.5rem; ...

The Rendering Pipeline

Understanding the flow helps clarify how everything connects:

  1. You write JSX ‚Äî React components with HTML-like syntax and Tailwind classes
  2. React compiles JSX ‚Äî Transforms your components into JavaScript function calls
  3. React creates Virtual DOM ‚Äî Builds a tree of React elements in memory
  4. React reconciles changes ‚Äî Compares new Virtual DOM with previous state
  5. Browser receives HTML + CSS ‚Äî React updates the actual DOM with standard HTML elements and CSS classes
  6. Browser renders ‚Äî The final output is standard HTML, CSS, and JavaScript that any browser can display

Tailwind & MUI ‚Äî Styling Smarter

Tailwind CSS is a utility-first CSS framework that generates CSS classes based on your usage. When you write className="flex items-center gap-4" in React, Tailwind's build process generates the corresponding CSS rules. The classes are applied to the HTML elements React renders, creating a seamless integration between React's component model and Tailwind's utility system.

Material UI (MUI) is a React component library that provides pre-built, styled components. Under the hood, MUI components are React components that render standard HTML elements with CSS (and optionally, CSS-in-JS). You can use MUI components alongside Tailwind, or use MUI's built-in styling system. Both approaches ultimately produce standard HTML and CSS.

The key insight: React, Tailwind, and MUI are all abstraction layers that make development faster and more maintainable, but they all compile down to the same web fundamentals: HTML for structure, CSS for styling, and JavaScript for behavior. Your foundational knowledge of HTML and CSS remains essential and directly applicable.

üí° In short: React helps you structure your site, Tailwind and MUI help you style it ‚Äî and together they let you focus on creating something meaningful rather than rebuilding what already exists.
