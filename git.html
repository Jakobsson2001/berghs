Quick start

Quick start

  1. Install Git and create a GitHub account.
  2. Create a repository.
  3. Run these in your terminal:
git clone <repo-url>
cd <repo>
# edit files
git add .
git commit -m "feat: first commit"
git push origin main

Tip: Use branches for new work, then open a Pull Request.

What is GitHub?

GitHub is an online platform for hosting git repositories. It adds collaboration tools (Issues, Pull Requests, code review), automation (Actions), and discovery (open-source) on top of git‚Äôs version history.

Core idea: keep every change to your code with a permanent history, and use branches to experiment safely.

Why teams use it

  • History
  • Collaboration
  • Automation (Actions)
  • Issues
  • Security (branch protection, CODEOWNERS, secret scanning)

How GitHub works

Repositories

Stores your code + history. Public or private. Default branch often main.

Branches

Develop features without breaking main. Merge via PR.

Pull Requests

Propose changes, show diffs, run checks, collect feedback.

Issues

Track work; link PRs so progress auto-updates.

Actions

CI/CD via YAML workflows that run on pushes/PRs.

Forks

Copy a repo to your account to propose changes to projects you don‚Äôt own.

Key concepts & terms

  • Commit: a saved snapshot of your work.
  • Clone: download a remote repository to your machine.
  • Remote (origin): a pointer to the hosted repo on GitHub.
  • Merge: combine branch changes preserving history.
  • Rebase: replay commits on top of another branch.
  • .gitignore: tells git which paths not to track.

Typical team workflow

  1. Pick issue ‚Üí create branch from main.
  2. Commit small, focused changes with clear messages.
  3. Push branch and open a PR.
  4. CI checks run via Actions.
  5. Get review, address comments, and merge when approved.
  6. Delete branch; PR links back to the issue automatically.

Git vs GitHub

Git = local version control tool (offline OK).

GitHub = hosted platform around git (collab, reviews, automation).
