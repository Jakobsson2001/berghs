CSS: Cascading Style Sheets

CSS is the language that transforms plain HTML into beautiful, styled web pages. Think of HTML as the skeleton of a webpage—it defines the structure. CSS is like the skin, clothes, and makeup—it controls how everything looks, from colors and fonts to spacing and layout.

How CSS Works

CSS works by applying rules to HTML elements. Each rule has three parts:

  • Selector — tells the browser which elements to style (e.g., p, .class-name, #id-name)
  • Property — what aspect to change (e.g., color, margin, font-size)
  • Value — the specific setting (e.g., blue, 16px, Arial)
/* Example CSS rule */
p {
  color: blue;
  font-size: 16px;
  margin: 10px;
}

The Box Model

Every HTML element is treated as a rectangular box with four layers:

  1. Content — the actual text or image
  2. Padding — space inside the element, around the content
  3. Border — a line around the padding
  4. Margin — space outside the element, between elements

Selectors: Targeting Elements

CSS selectors let you choose which elements to style. Here are the most common types:

Element Selector

p {
  color: red;
}
/* Styles all <p> elements */

Class Selector

.highlight {
  background-color: yellow;
}
/* Styles elements with class="highlight" */

ID Selector

#header {
  font-size: 24px;
}
/* Styles the element with id="header" */

Descendant Selector

div p {
  color: blue;
}
/* Styles <p> elements inside <div> elements */

Specificity: When Rules Conflict

When multiple CSS rules target the same element, specificity determines which one wins. The hierarchy is:

  1. Inline styles (highest priority) — style="color: red;"
  2. ID selectors — #my-id
  3. Class selectors — .my-class
  4. Element selectors (lowest priority) — p

If two rules have the same specificity, the one that appears later in the stylesheet wins.

Inheritance

Some CSS properties are inherited by child elements, while others are not:

  • Inherited: color, font-family, font-size, line-height
  • Not inherited: margin, padding, border, background

This means you can set a base style on the <body> element, and all text inside will inherit that color and font.

Layout with Flexbox

Flexbox is a powerful layout system that makes it easy to align and distribute space among items. It's perfect for creating flexible, responsive layouts.

.container {
  display: flex;
  justify-content: center;  /* horizontal alignment */
  align-items: center;      /* vertical alignment */
  gap: 20px;                /* space between items */
}

Layout with CSS Grid

CSS Grid is ideal for creating two-dimensional layouts with rows and columns. It gives you precise control over both dimensions.

.grid-container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

Responsive Design with Media Queries

Media queries let you apply different styles based on screen size, making your website work on phones, tablets, and desktops.

/* Default styles for mobile */
.container {
  padding: 10px;
}

/* Styles for tablets and larger */
@media (min-width: 768px) {
  .container {
    padding: 20px;
  }
}

/* Styles for desktops */
@media (min-width: 1024px) {
  .container {
    padding: 40px;
    max-width: 1200px;
  }
}

CSS Variables (Custom Properties)

CSS variables let you store values that you can reuse throughout your stylesheet. Change the variable once, and it updates everywhere.

:root {
  --primary-color: #e91e63;
  --spacing: 16px;
}

.button {
  background-color: var(--primary-color);
  padding: var(--spacing);
}

Transitions and Animations

CSS can create smooth transitions and animations without JavaScript. Use transition for simple changes and @keyframes for complex animations.

/* Simple transition */
.button {
  background-color: blue;
  transition: background-color 0.3s ease;
}

.button:hover {
  background-color: red;
}

/* Animation */
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

.fade-in {
  animation: fadeIn 1s ease-in;
}

Common CSS Properties

Here are some essential properties you'll use frequently:

  • color — text color
  • background-color — background color
  • font-size — text size
  • font-family — font type
  • margin — outer spacing
  • padding — inner spacing
  • border — border around element
  • border-radius — rounded corners
  • display — how element is displayed (block, inline, flex, grid)
  • width and height — element dimensions

Best Practices

  • Use classes, not IDs — Classes are reusable and have lower specificity, making them easier to override
  • Keep selectors simple — Avoid overly specific selectors like div.container > ul > li
  • Use meaningful class names — .card is better than .box1
  • Organize your CSS — Group related rules together (typography, layout, colors, etc.)
  • Use CSS variables — Store colors, spacing, and other values in variables for easy maintenance
  • Test in multiple browsers — Use browser developer tools to preview and debug
  • Start mobile-first — Design for small screens first, then add styles for larger screens

Browser Developer Tools

Every modern browser includes developer tools that let you:

  • Inspect elements and see their CSS
  • Edit CSS live and see changes instantly
  • Test different screen sizes
  • Debug layout issues

Press F12 or right-click and select "Inspect" to open developer tools.
