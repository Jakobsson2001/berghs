Introduction to CSS

Cascading Style Sheets, commonly shortened to CSS, is the language that designers and developers use to decide how web pages look and feel. When you visit a site, the browser reads the HTML to understand the content, and CSS to figure out how that content should be presented. Think of HTML as the structure of a house, while CSS is the paint, furniture, and decorations that make the rooms inviting. By adjusting colors, sizes, positioning, and effects, CSS turns documents into polished experiences.

CSS works by applying rules. A rule tells the browser which part of the page to target and which visual instructions to follow. The part being targeted is called the selector. It might be an element name such as paragraph, a class assigned to several elements, or a unique identifier used once on the page. The instructions are written as property and value pairs. For example, the property color set to blue tells the browser to display the selected text in blue.

Because many rules can apply to the same element, CSS uses a system called the cascade to decide which rule wins. Specificity, source order, and whether a rule is marked as important all influence this decision. A rule that targets a specific element, such as using an identifier, is more powerful than a general rule that targets all paragraphs. If two rules have identical strength, the one that appears later in the style sheet takes priority. Understanding this cascade prevents unexpected results and helps you organize your styles.

CSS also controls layout, which is how elements flow and align on a page. Early designers relied on tables and complicated tricks, but modern CSS provides dedicated tools. The box model describes every element as a rectangular box made of content, padding, borders, and margins. Display values, such as block, inline, and inline-block, determine how these boxes behave with each other. Layout modules like Flexbox and Grid offer ways to distribute space, create columns, and adapt designs to different screen sizes without extra code.

Responsive design is the practice of making a site look good on phones, tablets, laptops, and large screens. CSS enables this by letting you write media queries, which are conditions that check the size or characteristics of the device. When a condition is met, the browser uses the matching styles. You can increase font sizes on small screens, rearrange columns into stacks, or hide content that might clutter a display. Responsive CSS ensures that everyone enjoys the website without needing a separate mobile version.

An aspect of CSS is inheritance, where properties pass from parent elements to their children. For instance, if you set the text color of the body element, paragraphs inside adopt that color. Not every property inherits, so you can choose when to override defaults. Combined with classes, inheritance helps keep sheets shorter and easier to maintain. Instead of repeating the color or font rule everywhere, you define it once and let the cascade do the rest.
